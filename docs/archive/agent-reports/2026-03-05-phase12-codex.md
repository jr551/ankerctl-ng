## Key Behaviors
- Apprise client posts JSON payloads to configured endpoint (`server_url` + `/notify/{key}`) with:
  - `title`
  - `body`
  - `type` (`info|success|failure`)
  - optional `tag`
  - optional `attach` (base64 data-URI list)
- Event support includes:
  - `print_started`
  - `print_finished`
  - `print_failed`
  - `print_paused`
  - `print_resumed`
  - (existing compatibility retained: `gcode_uploaded`, `print_progress`)
- Templates for overlapping events match Python defaults exactly.
- Snapshot attachment:
  - If `APPRISE_ATTACH=true`, snapshot is captured via `VideoQueue.CaptureSnapshot(...)`
  - Attached as base64 data URI (`data:image/jpeg;base64,...`)
- Notification service:
  - Subscribes to MQTT stream via `Tap`
  - Tracks print-state transitions from `event=print_state`
  - Emits started/paused/resumed/finished/failed events accordingly
  - Respects `context.Context` in runtime loop and HTTP requests
- `/api/notifications/test` now sends a real test notification (with optional payload override and optional snapshot attachment)
