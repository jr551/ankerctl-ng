## Decisions Made
- Implemented dedicated WebSocket handlers under `internal/web/ws` and wired them directly in `internal/web/routes.go`.
- Used gorilla/websocket read/write pump separation with ping/pong keepalive (`30s` ping, `60s` pong wait).
- For stream handlers (`/ws/mqtt`, `/ws/video`, `/ws/upload`), used buffered intermediary channels (`size 32`) and non-blocking Tap callbacks to avoid callback backpressure.
- `/ws/ctrl` enforces inline first-message auth (`api_key`) with `crypto/subtle.ConstantTimeCompare`, and closes with code `1008` on policy violations.
- `/ws/ctrl` command contract implemented as requested:
  - `{"cmd":"light","value":"on|off"}`
  - `{"cmd":"video_profile","value":"sd|hd|fhd"}`
  - `{"cmd":"video_enable","value":true|false}`
- Added `Server.VideoSupported()` so WS handlers can enforce printer camera capability checks.
- Added `PPPPService.IsConnected()` for `/ws/pppp-state` polling.
- Added table-driven GCode tests covering `PatchGCodeTime` and `ExtractLayerCount` scenarios from spec.

## Deviations from Spec
- `/ws/pppp-state` emits an initial status immediately on connect and then every 2 seconds; payload includes `service_state` when `ppppservice` is registered. This is additive and backward compatible.
